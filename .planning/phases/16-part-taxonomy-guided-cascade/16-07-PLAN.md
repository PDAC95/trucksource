<objective>
Let sellers declare which truck years a listing fits, and show it to buyers — the create/edit half of the YEAR feature (data + RPC shipped in 16-05).

Seller UX (stakeholder-confirmed): a Year Compatibility section with a mode toggle — **Universal (all years)** (default), **Specific year** (one select), or **Year range** (start + end selects). Specific stores `year_start = year_end`; range stores the pair; universal stores both null. Years come from `yearOptions()` (16-05). The SAME Zod schema validates on the client (UX) and re-validates inside the Server Action (trust boundary), per CLAUDE.md. The edit form reuses the schema/form and pre-fills the mode + years from the stored row. The public listing detail renders the compatibility ("Fits years: 2008–2015" / "2012" / nothing if universal).

Output: year fields in `schema.ts`, a Year section in `listing-form.tsx`, persistence in `actions/listings.ts`, selection + mapping in `lib/listings/queries.ts`, display in `listing-detail.tsx`, and edit pre-fill in `sell/[id]/edit/page.tsx`. All UI copy ENGLISH.
</objective>
