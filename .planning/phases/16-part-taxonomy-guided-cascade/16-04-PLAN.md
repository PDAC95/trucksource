<objective>
Replace the flat single-select Category facet on `/browse` with three dependent selects — **Category(root) -> Subcategory -> Item** — mirroring the existing Make -> Model -> Config cascade idiom in `FacetControls`. Because `browse-toolbar-mobile.tsx` renders the SAME `FacetControls` body, one edit covers desktop sidebar and mobile sheet.

The URL contract keeps a SINGLE `category` id (the subtree-match RPC from Plan 01 expands it), so the three selects resolve to ONE `category` param = the deepest chosen node. Two helper params (`subcategory`, `item`) are kept in the URL ONLY to remember which selects to show as chosen on reload and to drive dependent loading — but `category` always equals the deepest of the three. The page resolves the chip label to show ancestor context so a shared URL is unambiguous (Pitfall 6).

Purpose: the buyer can drill the taxonomy as URL-shareable facets on the results page, and ancestor selections find all leaf-tagged descendants via the subtree match.
Output: reworked `FacetControls` category section, page wiring for root categories + chip-label resolution, and chip-removal coverage for the new levels.
</objective>
