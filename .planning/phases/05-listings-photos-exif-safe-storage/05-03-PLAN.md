  <name>Task 1: Photo upload + remove Server Actions (the EXIF wiring)</name>
  <files>lib/actions/listings.ts</files>
  <action>
Create `lib/actions/listings.ts` with `"use server";`. Header comment mirroring garage.ts: identity via getClaims (NEVER getSession), cookie-bound user client so Storage + table RLS is the authorization boundary, NO service-role client.

`uploadListingPhoto(form: FormData): Promise<UploadPhotoResult>` per 05-RESEARCH Pattern 2:
1. `createClient()` → `getClaims()` → `userId = claims?.claims?.sub`; if missing → `unauthenticated`.
2. `const file = form.get("file"); if (!(file instanceof File)) return { ok:false, error:"invalid" };`
3. `const input = Buffer.from(await file.arrayBuffer());`
4. `const stripped = await stripAndReencode(input, file.type);` — if `!stripped.ok`, return `{ ok:false, error: stripped.error }` (maps too_large/unsupported_type/decode_failed straight through; HEIC lands here as unsupported_type).
5. Path = `${userId}/staging/${crypto.randomUUID()}.${stripped.ext}` (Pattern 3 per-user staging prefix).
6. `supabase.storage.from(LISTING_PHOTOS_BUCKET).upload(path, stripped.buffer, { contentType: stripped.contentType, cacheControl: "3600", upsert: false })`; on error → `upload_failed`.
7. Return `{ ok:true, path }`. Only the re-encoded buffer is ever uploaded — the original (with GPS) is NEVER persisted (anti-pattern).

`removeListingPhoto(path: string): Promise<{ ok: boolean }>`:
- getClaims; guard that `path.startsWith(`${userId}/`)` (don't delete outside own folder); `supabase.storage.from(LISTING_PHOTOS_BUCKET).remove([path])`. Storage RLS also enforces this; the app check is belt-and-suspenders. Used by the uploader to drop a just-uploaded staged photo the seller removed before publish.

Keep the strip server-only: sharp is only reachable through this action (it imports `lib/images/strip` which is `import "server-only"`).
