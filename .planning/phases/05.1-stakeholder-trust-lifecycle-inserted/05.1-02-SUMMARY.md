- `updateSellerType` + `updateDisplayName` Server Actions cloned from `updateContactPreference` — getClaims identity (never getSession), same-schema re-validation at the trust boundary, owner-RLS write through the cookie client (no service-role), zero-rows→`not_found`. `updateDisplayName` returns the resolved `publicName` for a no-second-read preview toast.

- `/account` now renders three controls: the deliberate public-name reveal/revert, the seller-type select, and the existing contact preference.
- The public profile (`/u/[username]`) shows `coalesce(display_name, username)` as the heading with the seller-type badge beside it, while staying anon-safe and PII-free.

## Decisions Made
- `updateDisplayName` returns `publicName` so the preview toast ("Your public name is now: X") needs no extra round-trip.
- The reveal WRITE fires only on the modal Confirm; the form submit merely opens the modal — making the identity reveal a deliberate two-step action (ACCT-08 CONTEXT lock).
- Revert writes `display_name = null` and never touches `username`, so the original anonymous handle structurally returns (Pitfall 2).
