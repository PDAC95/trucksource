1. **`public.listing_comments`** — `id bigint identity pk`, `listing_id bigint not null references public.listings(id) on delete cascade`, `author_id uuid not null references auth.users(id) on delete cascade`, `parent_id bigint references public.listing_comments(id) on delete cascade`, `body text not null check (char_length(body) between 1 and 1000)`, `created_at timestamptz not null default now()`. Indexes: `(listing_id, created_at desc)` and `(parent_id)`. RLS policies:
   - SELECT to `anon, authenticated` using (true) — public-read.
   - INSERT to `authenticated` with check: `(select auth.uid()) = author_id` AND listing is ACTIVE and unexpired (`exists` subquery on `public.listings` with `status = 'active' and (expires_at is null or expires_at > now())` — this is the structural "comments closed when sold" rule, Pitfall 2) AND (`parent_id is null` OR parent exists with `p.listing_id = listing_id and p.parent_id is null` — structural depth-1 + same-listing graft guard, Pitfall 3).
   - DELETE to `authenticated` using: own comment (`auth.uid() = author_id`) OR caller sells the listing (`exists` on `public.listings` with `seller_id = (select auth.uid())`).
   - NO update policy (locked: no editing — the absence IS the enforcement). Scope auth.uid() as `(select auth.uid())` everywhere.

2. **`public.saved_listings`** — composite pk `(user_id, listing_id)`, `user_id uuid not null references auth.users(id) on delete cascade`, `listing_id bigint not null references public.listings(id) on delete cascade`, `created_at timestamptz default now()`, index on `(listing_id)`. RLS: owner-only SELECT/INSERT/DELETE to `authenticated` with `(select auth.uid()) = user_id`. NO anon policy of any kind.

3. **`public.comment_deletion_log`** — columns per Pattern 4 (comment_id, listing_id, author_id, parent_id, body snapshot, comment_created_at, deleted_by uuid, deleted_at default now()). `enable row level security` with ZERO policies — default-deny; only the definer trigger writes, only service role (Phase 10) reads. Mirrors listing_view_events posture.

4. **`public.log_comment_deletion()`** — `returns trigger language plpgsql security definer set search_path = ''`, inserts the OLD row snapshot + `(select auth.uid())` as deleted_by. Then `create trigger listing_comments_audit before delete on public.listing_comments for each row execute function public.log_comment_deletion();`. Row triggers fire per FK-cascaded reply delete, so replies are audited too.

5. **`public.my_listing_save_counts()`** — `returns table (listing_id bigint, save_count bigint) language sql stable security definer set search_path = ''`, body per Pattern 2 (join saved_listings→listings, `where l.seller_id = (select auth.uid())`, group by). Then `revoke all on function public.my_listing_save_counts() from public, anon; grant execute ... to authenticated;`. Copy the 0008 active_listing_count definer hygiene verbatim — schema-qualify EVERYTHING (search_path is empty).

Make the migration idempotent where cheap (`create table if not exists` is NOT needed — this is a fresh migration — but use `add column if not exists` for the listings alter and `create or replace function` for functions; drop-if-exists the trigger before create).

6. **No update path:** authenticated update of own comment body is rejected/affects 0 rows (no UPDATE policy).
7. **Saves owner-only:** user A inserts a save (succeeds, self user_id); user A select sees it; anon select returns 0 rows/denied; insert with another user_id rejected. Clean up.
8. **Audit default-deny:** anon AND authenticated `select` on `comment_deletion_log` returns error or zero visibility; authenticated insert rejected.
9. **Audit trigger fires:** as the comment author, delete the parent comment from gate 5 — then (since clients can't read the log) assert indirectly: the delete succeeded and the reply is gone too (cascade). The log-row presence itself is service-role territory; if `_supabase.ts` exposes a service-role client for tests, additionally assert 2 audit rows (parent + cascaded reply) with `deleted_by` = author uid. If no service-role test client exists, document the cascade assertion as the gate and leave the log-row check to the success-criteria sweep.
