Create `supabase/migrations/0004_garage.sql` in the exact lowercase-SQL style of `0003_fitment_taxonomy.sql`. Define:

```
create table public.garage_trucks (
  id bigint generated always as identity primary key,
  user_id uuid not null references auth.users(id) on delete cascade,
  model_id bigint not null references public.models(id) on delete restrict,
  config_id bigint references public.configurations(id) on delete restrict,  -- NULL = model-level
  nickname text,
  created_at timestamptz not null default now(),
  constraint nickname_len check (nickname is null or char_length(nickname) <= 40)
);
create index garage_trucks_user_id_idx on public.garage_trucks (user_id);
-- No-exact-duplicate (Make,Model,Config) per user. coalesce folds the NULL config arm
-- (constraints can't use expressions; must be a unique INDEX — the 0003 search_term_targets_uniq trick).
create unique index garage_trucks_uniq on public.garage_trucks (user_id, model_id, coalesce(config_id, 0));

alter table public.garage_trucks enable row level security;
```
Then the FOUR owner policies, all `to authenticated`, all using the `(select auth.uid())` wrapper (perf-correct, repo-wide convention):
- select: `using ((select auth.uid()) = user_id)`
- insert: `with check ((select auth.uid()) = user_id)`
- update: `using ((select auth.uid()) = user_id) with check ((select auth.uid()) = user_id)`
- delete: `using ((select auth.uid()) = user_id)`

CRITICAL — do NOT add any anon policy and do NOT add any `to anon` grant: garage must be structurally invisible publicly (CONTEXT/CLAUDE invariant). Do NOT store `make_id` (derive via models.make_id). Do NOT use SECURITY DEFINER anywhere here (private owner data — a DEFINER read would bypass RLS and leak other users' garages).

import { z } from "zod";
export const truckSchema = z.object({
  modelId: z.coerce.number().int().positive(),                       // Make+Model required (make implied by model)
