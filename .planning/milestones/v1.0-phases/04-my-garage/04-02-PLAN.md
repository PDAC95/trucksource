Define the result union and `addTruck`:
```ts
export type AddTruckResult =
  | { ok: true; id: number }
  | { ok: false; error: "unauthenticated" | "invalid" | "invalid_combo" | "duplicate" | "cap_reached" };

export async function addTruck(input: unknown): Promise<AddTruckResult> { ... }
```
Order inside addTruck:
1. `createClient()` → `getClaims()` → `userId = claims?.claims?.sub`; if absent return `unauthenticated`.
2. `truckSchema.safeParse(input)`; if `!success` return `invalid`. Destructure `{ modelId, configId, nickname }`. Normalize `nickname || null` (empty-string ⇒ NULL, Pitfall 5).
3. SOFT CAP (~20): `const { count } = await supabase.from("garage_trucks").select("id", { count: "exact", head: true });` (RLS scopes the count to the owner). If `(count ?? 0) >= 20` return `cap_reached`. Use a `const GARAGE_SOFT_CAP = 20;` module constant so the number is tunable (Claude's discretion per CONTEXT).
4. COMBO RE-CHECK (only when `configId != null`): query `model_configurations` for a row with `model_id = modelId AND configuration_id = configId` via `.maybeSingle()`; if none return `invalid_combo` (drives the "Missing your truck? Let us know" affordance).
5. INSERT `{ user_id: userId, model_id: modelId, config_id: configId ?? null, nickname: nickname || null }`, `.select("id").single()`. On error: if `error.code === "23505"` return `duplicate`; else return `invalid`. On success return `{ ok: true, id: data.id }`.

Do NOT trust the client for combo validity, cap, or user_id — the RLS `with check` also enforces `user_id`, but set it explicitly. Do NOT use the service-role/admin client anywhere (this is owner data through the cookie client).
