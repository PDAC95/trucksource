<seed_rules>
From RESEARCH Pattern 5 + Pitfall 2 (LOCKED):
- Parents before children: makes → models → configurations → model_configurations → search_terms → search_term_targets; flat L5–L8 in any order.
- Resolve every FK by NATURAL KEY inside the insert (`join public.makes m on m.name = ...`). NEVER write a literal generated id — ids differ between a fresh local reset and Staging (the #1 seed-portability bug).
- Idempotent: every insert uses `on conflict (<the unique constraint authored in 0003>) do nothing`. The constraints exist: makes.name, models(make_id,name), configurations.name, model_configurations(model_id,configuration_id), search_terms.term (citext), part_categories(parent_id,name), materials/conditions/special_filters.name, and the search_term_targets_uniq index.
- DO NOT seed explicit typo/misspelling rows ('Areodyne','flatglass') — typo tolerance is pg_trgm in Phase 7 (RESEARCH Open Q2). If a deliberate ALTERNATE spelling is a real term truckers type, seed it as a normal search_terms row pointing at the same entity (the M2M arc supports it).
- NO sentinel "Barnyard" category row — Barnyard is a boolean on listings (Phase 5).
</seed_rules>
