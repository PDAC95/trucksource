# Phase 05.1 Plan 04: Same-Seller Duplicate Warning (LIST-10) Summary

A non-blocking, advisory same-seller duplicate-title warning: on a publish attempt the create form probes the seller's own non-sold listings via the `find_similar_own_listings` trigram RPC (threshold 0.6), and when similar listings exist shows a non-accusatory dialog with edit-links plus a one-click "Publish anyway" — the probe is a separate advisory action that never blocks publish and leaves `createListing` completely unchanged.

## What Shipped

**Task 1 — advisory probe + gate-(e) test (commit `ff0c433`):**
- `lib/listings/duplicates.ts`: `findSimilarOwnListings(title)` — `getClaims` identity (returns `[]` if unauthenticated, no RPC call), calls `supabase.rpc("find_similar_own_listings", { p_title, p_threshold: 0.6 })`, maps rows to `{ id, title }[]` (drops `sim` from the UI payload). On RPC error it returns `[]` — a probe failure degrades to "no warning", never throws, never blocks. The RPC is SECURITY INVOKER + owner-scoped, so it only ever returns the caller's own non-sold listings, and it's backed by the `listings_title_trgm_idx` gin index (no seq scan, CLAUDE.md invariant #7). Threshold is the named const `DUPLICATE_SIMILARITY_THRESHOLD = 0.6`, passed to the RPC as a parameter (retune with no migration).
- `tests/unit/duplicate-probe.test.ts` (mocked client): proves gate (e) — `[]` when unauthenticated (no RPC call); mapped `{id,title}[]` when the RPC returns rows; `[]` when the RPC errors. A source-inspection assertion (comment-stripped) proves the probe module neither imports nor calls `createListing` — it cannot block publish by construction.

**Task 2 — warning dialog + create-form wiring (commit `d29cba6`):**
- `components/listings/duplicate-warning.tsx`: a controlled shadcn `Dialog` driven by `{ open, matches, onPublishAnyway, onCancel }`. Non-accusatory copy ("If this is the same part, you can edit the original. If it's a different part, go right ahead."), each match rendered as a link to `/sell/{id}/edit`, and two actions — "Publish anyway" (primary) and "Go back" (dismiss = cancel, also fired on overlay/esc/X). Never a blocking-only state.
- `components/listings/listing-form.tsx`: wired the probe into the CREATE path only. New state `duplicateMatches` / `pendingPayload` / `showDuplicateDialog`. In `onSubmit`, on the create path, BEFORE calling `createListing`, it awaits `findSimilarOwnListings(payload.title)`; if matches return it stashes the payload and opens the dialog (no publish yet). `onPublishAnyway` closes the dialog and calls the UNCHANGED `createListing` via a shared `runCreate(payload)`. No matches (or a probe failure → `[]`) falls straight through to `runCreate`. The edit branch and `createListing` are untouched; the existing `onInvalid` toast + RHF flow are intact.
